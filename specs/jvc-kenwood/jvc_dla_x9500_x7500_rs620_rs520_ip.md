# JVC KENWOOD DLA-X9500/X7500/RS620/RS520 Control Spec

## Summary
JVC KENWOOD D-ILA projector series supporting both RS-232C and TCP/IP LAN control. Direct commands control power, input switching, gamma, lamp power, test patterns, and picture settings. Remote control emulation commands mirror the physical remote. LAN control uses a challenge-response handshake on port 20554 before sending standard hex commands.

<!-- UNRESOLVED: RS-232C and LAN control supported; infrared control not represented in this spec -->

## Transport
```yaml
protocols:
  - tcp
  - serial
addressing:
  port: 20554  # stated: LAN control port
serial:
  baud_rate: 19200  # stated: RS-232C 19.2kbps
  data_bits: 8
  parity: none
  stop_bits: 1
  flow_control: none
auth:
  type: none  # inferred: no auth procedure in source; LAN uses challenge-response handshake
```

## Traits
```yaml
- powerable       # power on/off direct commands present
- routable        # input switching commands present
- queryable       # enquiry commands for power, input, gamma, source status present
- levelable       # brightness, contrast, colour, tint, sharpness adjustment commands present
```

## Notes

**Command protocol — RS-232C and LAN share same hex command set.** Commands are 7 or 10 bytes: header (21 for operating, 3F for enquiry) + unit ID (89 01) + command (2 bytes) + data (variable) + end (0A). Acknowledgement responses are 06 89 01 followed by first 2 bytes of command, then 0A.

**LAN handshake sequence:** TCP connection → projector sends `PJ_OK` → controller sends `PJREQ` within 5s → projector sends `PJACK` → controller sends command within 5s → projector closes connection after 5s. Port: 20554.

**RS-232C physical:** 9-pin D-Sub male; Tx=pin 3, Rx=pin 2, Ground=pin 5. Config: 19200 bps, 8N1, no flow control. Binary format.

**Power off note:** Source states "Power - Off (send twice with short delay between to switch off)" — this is a direct command requirement, not an emulation command.

**Error handling:** Projector ignores unrecognized commands (bad Unit ID, parity error, invalid command) and inappropriate commands (e.g., Power On during cooling). Discards commands if gap ≥50ms in incoming data. External controller must wait for acknowledgement before sending next command.
